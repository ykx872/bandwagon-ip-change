## 先确认：IP 是不是真的被封了？

别急着花钱换 IP。有时候连不上只是机器暂时挂了，或者是 VPS 的进程跑死了。

搬瓦工在 KiwiVM 面板里自带了一个 IP 封锁检测工具。进 KiwiVM 控制台之后，找到左侧菜单里的 **Main IP Blacklist Check**，点进去点 **Test Main IP**，结果有两种：

- **IP NOT BLOCKED**：IP 正常，问题出在别处，去查查 SSH 端口、防火墙之类的

这一步一定要先做，因为 IP 一旦被封，KiwiVM 的机房迁移功能会自动被锁住——换句话说，被封之后你会发现"迁移到其他数据中心"这个操作根本点不了。这是官方政策，不是 Bug。

## 换完IP之后别忘了这几件事

很多人换完IP就以为结束了，结果发现还是有问题，通常是没做好这几步：

1. **更新域名DNS解析**：如果网站绑定了旧IP，记得去域名服务商把A记录改成新IP，解析生效需要一点时间
2. **刷新本地DNS缓存**：电脑/路由器有时会缓存旧IP，试试 `ipconfig /flushdns`（Windows）或重启网络
3. **检查服务器上的防火墙规则**：有些iptables规则是针对旧IP写的，要核对一下
4. **备份数据**：其实在换IP之前备份更好，官方说明写着数据不会丢失，但重要数据还是先备一份稳妥

## 换IP还是直接换机房？

这个问题问的人挺多。说实话，两条路各有适用场景：

**付费换IP（$8.79）**：适合想留在当前机房线路、数据和配置都不想动的情况。操作简单，基本是最省事的选项。

## 常见问题

**Q：提交更换IP工单后，付款前可以取消吗？**

A：可以。点了 Request IP Change 之后会生成工单和账单，但只要不付款，什么都不会发生。账单就挂在那里，不付就不换，也不会自动扣费。

**Q：搬瓦工更换IP工单要多久处理完？**

A：官方承诺24小时内完成。实际用下来多数都在几分钟到1小时内搞定，很少要等满24小时的。

**Q：新换的IP会不会也被封？**

A：搬瓦工在换IP之前会验证新IP在全球各地的可达性，确认没问题再分配给你。但如果后续使用方式没变，不排除再次被封的可能。

**Q：换IP之后，旧IP怎么了？**

A：旧IP会被回收进搬瓦工的IP池，官方会再次确认后才重新分配给其他用户。你的服务器数据全程保留，不受影响。

**Q：IP被封了，能不能通过切机房免费换IP？**

A：不行。IP一旦被官方检测到已封锁，KiwiVM 的机房迁移功能会自动被禁用，必须先通过付费换IP解决，才能恢复迁移权限。

**Q：搬瓦工有哪个套餐可以免费换IP？**

A：E-Commerce SLA 系列套餐自带每两周免费换一次IP的功能，是目前唯一有这个政策的套餐线。
